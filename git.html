Guia Básico de Git e GitHub

O que é Git?

Git é um sistema de controle de versão distribuído que permite que desenvolvedores trabalhem simultaneamente no mesmo projeto. Ele rastreia as alterações no código-fonte, facilitando o gerenciamento e a colaboração.

Comandos Essenciais do Git

1. Inicializar um repositório

Crie um novo repositório Git em seu diretório de projeto: 

git init

2. Clonar um repositório

Faça o download de um repositório remoto para seu computador local: 

git clone <url-do-repositorio>

3. Verificar o status

Veja quais arquivos foram modificados e quais estão prontos para serem commitados: 

git status

4. Adicionar arquivos ao staging

Adicione arquivos para o próximo commit: 

git add <nome-do-arquivo>

5. Commitar alterações

Salve suas alterações com uma mensagem descritiva: 

git commit -m "Mensagem descritiva do commit"
Exemplo:
git commit -m "Adiciona função de login e validação"

6. Enviar alterações para o repositório remoto

Envie seus commits para o repositório remoto (no GitHub, por exemplo): 

git push origin main

O que é GitHub?

GitHub é uma plataforma que hospeda repositórios Git na nuvem. Ele facilita a colaboração, o compartilhamento de código e a contribuição para projetos open-source.

Trabalhando com GitHub

1. Criar um repositório remoto

No GitHub, clique em "New repository" para criar um novo repositório. Copie a URL fornecida para clonar ou adicionar ao seu repositório local.

2. Associar um repositório local a um remoto

Adicione a URL do repositório remoto ao seu projeto local: 

git remote add origin <url-do-repositorio>

3. Fazer um pull das alterações

Baixe alterações do repositório remoto para sua máquina local: 

git pull origin main

Dicas de Boas Práticas

- Use mensagens de commit claras e concisas.
- Crie branches para novas funcionalidades ou correções.
- Faça commits frequentes e significativos.
- Nunca faça "force push" a menos que tenha certeza.
