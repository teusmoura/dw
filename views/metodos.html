Guia Completo de Métodos HTTP

Este guia cobre os principais métodos HTTP usados para comunicação entre clientes e servidores. Ele explica como e quando usar cada método com exemplos práticos em JavaScript e APIs.

1. O que são Métodos HTTP?

HTTP (HyperText Transfer Protocol) é o protocolo usado na web para a comunicação entre clientes (navegadores) e servidores. Os **métodos HTTP** são usados para especificar a ação que o cliente deseja que o servidor execute. Cada método tem um propósito específico e é utilizado em diferentes situações ao lidar com APIs e requisições web.

2. Principais Métodos HTTP

Aqui estão os métodos HTTP mais comumente utilizados, com suas funções e exemplos práticos:

2.1 GET

O método GET é usado para solicitar dados de um servidor. Ele não altera o estado do servidor, pois seu objetivo é apenas recuperar informações. GET é o método padrão quando acessamos páginas da web.

Exemplo de Requisição GET

// Requisição GET para obter dados de uma API
fetch("https://api.exemplo.com/usuarios")
    .then(response => response.json())
    .then(data => console.log(data))
    .catch(error => console.error("Erro:", error));

No exemplo acima, o método fetch faz uma requisição GET para uma API de usuários e exibe os dados retornados.

2.2 POST

O método POST é usado para enviar dados ao servidor, normalmente para criar ou adicionar novos recursos. Diferente do GET, o POST pode alterar o estado do servidor.

Exemplo de Requisição POST

// Requisição POST para enviar dados a uma API
let usuario = {
    nome: "Maria",
    idade: 25
};

fetch("https://api.exemplo.com/usuarios", {
    method: "POST",
    headers: {
        "Content-Type": "application/json"
    },
    body: JSON.stringify(usuario)
})
.then(response => response.json())
.then(data => console.log("Usuário criado:", data))
.catch(error => console.error("Erro:", error));

No exemplo acima, enviamos um objeto usuario para o servidor via POST, criando um novo usuário na API.

2.3 PUT

O método PUT é usado para atualizar completamente um recurso existente no servidor. Ele substitui o recurso alvo pelos dados fornecidos na requisição.

Exemplo de Requisição PUT

// Requisição PUT para atualizar dados de um usuário
let usuarioAtualizado = {
    nome: "Maria",
    idade: 26
};

fetch("https://api.exemplo.com/usuarios/1", {
    method: "PUT",
    headers: {
        "Content-Type": "application/json"
    },
    body: JSON.stringify(usuarioAtualizado)
})
.then(response => response.json())
.then(data => console.log("Usuário atualizado:", data))
.catch(error => console.error("Erro:", error));

Nesse exemplo, usamos PUT para atualizar um usuário com ID 1, substituindo os dados antigos pelos novos fornecidos.

2.4 PATCH

O método PATCH é usado para fazer uma atualização parcial de um recurso existente. Ao contrário de PUT, ele não substitui todo o recurso, mas apenas os campos que são atualizados.

Exemplo de Requisição PATCH

// Requisição PATCH para atualizar parcialmente um usuário
let atualizacaoParcial = {
    idade: 27
};

fetch("https://api.exemplo.com/usuarios/1", {
    method: "PATCH",
    headers: {
        "Content-Type": "application/json"
    },
    body: JSON.stringify(atualizacaoParcial)
})
.then(response => response.json())
.then(data => console.log("Usuário parcialmente atualizado:", data))
.catch(error => console.error("Erro:", error));

No exemplo acima, apenas a idade do usuário foi alterada, sem modificar os outros campos.

2.5 DELETE

O método DELETE é usado para remover um recurso no servidor. Ele é geralmente utilizado para deletar dados específicos, como usuários ou posts, a partir de um ID fornecido.

Exemplo de Requisição DELETE

// Requisição DELETE para remover um usuário
fetch("https://api.exemplo.com/usuarios/1", {
    method: "DELETE"
})
.then(response => console.log("Usuário deletado"))
.catch(error => console.error("Erro:", error));

O exemplo acima deleta o recurso do usuário com ID 1 no servidor.

3. Diferenças entre PUT e PATCH

Embora ambos os métodos sejam usados para atualizar dados no servidor, há uma diferença importante entre PUT e PATCH:

  • PUT substitui completamente o recurso existente pelos novos dados enviados. Se um campo não for enviado na requisição, ele será removido do recurso.
  • PATCH faz atualizações parciais, alterando apenas os campos especificados na requisição, sem afetar os demais.

Use PUT quando precisar substituir todo o recurso, e PATCH quando precisar alterar apenas alguns campos específicos.

4. Métodos HTTP Menos Utilizados

Além dos métodos principais, existem alguns métodos HTTP que são menos utilizados, mas ainda assim úteis em casos específicos:

4.1 HEAD

O método HEAD é semelhante ao GET, mas ele retorna apenas os cabeçalhos da resposta, sem o corpo (o conteúdo). Isso é útil para verificar se um recurso existe ou obter informações sobre ele sem baixá-lo completamente.

Exemplo de Requisição HEAD

// Requisição HEAD para verificar se um recurso existe
fetch("https://api.exemplo.com/usuarios/1", {
    method: "HEAD"
})
.then(response => {
    if (response.ok) {
        console.log("Recurso encontrado");
    } else {
        console.log("Recurso não encontrado");
    }
});

4.2 OPTIONS

O método OPTIONS é usado para descobrir quais métodos HTTP o servidor aceita para um recurso específico. Ele é útil para pré-verificações de segurança (como CORS).

Exemplo de Requisição OPTIONS

// Requisição OPTIONS para descobrir os métodos permitidos
fetch("https://api.exemplo.com/usuarios", {
    method: "OPTIONS"
})
.then(response => console.log(response.headers.get("Allow")));

O cabeçalho Allow na resposta vai indicar quais métodos são permitidos, como GET, POST, etc.

5. Resumo dos Métodos HTTP

Método Descrição
GET Solicita dados do servidor. Não altera o estado do servidor.
POST Envia dados ao servidor para criar um novo recurso.
PUT Atualiza completamente um recurso existente.
PATCH Atualiza parcialmente um recurso existente.
DELETE Remove um recurso do servidor.
HEAD Recupera os cabeçalhos de uma resposta, sem o corpo.
OPTIONS Descobre os métodos HTTP permitidos para um recurso.

6. Comandos Úteis para Testar Métodos HTTP

fetch() - Fazendo Requisições HTTP com JavaScript

// Requisição GET
fetch("https://api.exemplo.com/usuarios")
    .then(response => response.json())
    .then(data => console.log(data));

// Requisição POST
fetch("https://api.exemplo.com/usuarios", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ nome: "Carlos", idade: 32 })
});

Testando Requisições com cURL (linha de comando)

// Requisição GET com cURL
curl -X GET "https://api.exemplo.com/usuarios"

// Requisição POST com cURL
curl -X POST "https://api.exemplo.com/usuarios" \
     -H "Content-Type: application/json" \
     -d '{"nome": "Maria", "idade": 28}'
