Git e GitHub

Este guia é para estudantes que estão começando a aprender Git e GitHub, com integração ao Visual Studio Code (VSCode). Aqui, você encontrará os principais recursos do Git, quando utilizá-los e como fazer isso diretamente no VSCode.

1. O que são Git e GitHub?

Git

Git é um sistema de controle de versão distribuído que permite rastrear alterações em arquivos e colaborar em projetos de software com outros desenvolvedores. É útil para gerenciar versões de código e garantir que você possa reverter ou compartilhar seu progresso com segurança.

GitHub

GitHub é uma plataforma online que hospeda repositórios Git, permitindo que você compartilhe seu código com outros desenvolvedores. Um repositório é como uma pasta que guarda todos os arquivos do seu projeto, além do histórico de alterações. O GitHub facilita o versionamento e a colaboração entre equipes.

2. Principais Comandos do Git

Comandos Básicos para Iniciantes

Aqui estão alguns dos comandos mais básicos que você vai usar ao começar com Git e GitHub.

git init - Inicializando um Repositório

Cria um novo repositório Git no diretório atual. O repositório é onde você vai armazenar todo o histórico do seu projeto.

git init

git add - Preparando Arquivos

Adiciona arquivos à staging area, preparando-os para o commit. A staging area é como um rascunho onde você coloca os arquivos que quer salvar no próximo commit.

git add .

git commit - Criando um Commit

Registra as mudanças feitas nos arquivos. Cada commit é como um "snapshot" do seu projeto em um momento específico, sempre acompanhado de uma mensagem descritiva para que você ou outros desenvolvedores saibam o que foi alterado.

git commit -m "Descrição clara do que foi alterado"

git status - Verificando o Status

Exibe informações sobre o estado atual do repositório, incluindo quais arquivos foram modificados e quais estão prontos para o commit.

git status

git log - Histórico de Commits

Exibe o histórico dos commits feitos no projeto, mostrando detalhes como a data e a mensagem de cada commit.

git log

3. Integração com GitHub

Conectando um Repositório Local ao GitHub

Para compartilhar seu código no GitHub, você precisa conectar o repositório local ao GitHub, criando um vínculo com um repositório remoto.

git remote add origin [URL]

Esse comando conecta o repositório local ao repositório remoto no GitHub.

git remote add origin https://github.com/seu-usuario/seu-repositorio.git

git push - Enviando para o GitHub

Envia seus commits do repositório local para o repositório remoto no GitHub. O comando push transfere o conteúdo das suas alterações para o GitHub.

git push origin main

git clone - Clonando um Repositório

Cria uma cópia de um repositório remoto (no GitHub, por exemplo) no seu computador, permitindo que você trabalhe localmente nesse projeto.

git clone https://github.com/seu-usuario/seu-repositorio.git

4. Fluxo de Trabalho no Git

1. Inicialize o repositório

git init

2. Faça modificações no seu código

git add .

3. Crie um commit com uma mensagem descritiva

git commit -m "Descrição das alterações"

4. Envie para o GitHub

git push origin main

5. Git no VSCode

Usando Git Diretamente no Visual Studio Code

Visualizando Alterações

Use o ícone de "controle de versão" na barra lateral do VSCode para visualizar alterações, arquivos modificados e criar commits. O VSCode mostra diretamente quais arquivos foram alterados e permite ver os detalhes dessas mudanças.

Adicionando Arquivos

Quando um arquivo for modificado, você pode clicar no ícone + ao lado do nome do arquivo para adicioná-lo ao staging.

Commitando Alterações

Digite a mensagem de commit diretamente no campo apropriado no VSCode e clique no ícone de check (✓) para salvar o commit.

Enviando para o GitHub

Após fazer o commit, você pode usar o ícone de "sincronizar" para fazer o push das mudanças para o GitHub.

6. Recursos Avançados do Git

6.1 Fork e Pull Requests

Se você quiser colaborar em um projeto no GitHub, você pode fazer um **fork**, que é basicamente uma cópia do repositório para o seu perfil. Depois de fazer mudanças na sua cópia, você pode enviar um **pull request** para sugerir que essas mudanças sejam incorporadas ao repositório original.

git remote add upstream https://github.com/original-user/repository.git

6.2 Trabalhando com Branches

Branches são "linhas do tempo" paralelas que permitem que você trabalhe em funcionalidades ou correções de forma isolada, sem afetar o código principal. Isso mantém o código principal estável enquanto você desenvolve.

git checkout -b nova-branch

6.3 Resolvendo Conflitos de Merge

Quando você mescla (merge) duas branches, pode ocorrer um conflito se ambas tiverem alterado a mesma parte de um arquivo. O Git ajuda você a resolver esses conflitos mostrando as diferenças para que você decida o que manter.

git add .
git commit -m "Conflito resolvido"

6.4 Git Rebase

O git rebase permite criar um histórico de commits mais linear, reaplicando commits de uma branch sobre outra, tornando o histórico mais limpo.

git rebase main

6.5 Desfazendo Mudanças

Se você cometeu um erro em um commit, pode desfazer essas alterações com git reset ou git revert. O git reset remove os commits, enquanto o git revert cria um novo commit que desfaz as mudanças do commit anterior.

git reset --soft HEAD~1
git revert [commit]

7. Comandos Úteis no Git

git pull - Atualizando o Repositório Local

O comando git pull baixa as atualizações feitas no repositório remoto e as combina com o repositório local.

git pull origin main

git branch - Gerenciando Branches

Cria ou lista branches existentes. As branches permitem que você trabalhe em diferentes partes de um projeto de forma separada.

git branch nome-da-branch

git diff - Visualizando Diferenças

Mostra as diferenças entre o estado atual do código e o último commit. Isso é útil para ver o que mudou antes de fazer um commit.

git diff HEAD

git add -p - Staging Parcial

Permite adicionar apenas partes específicas de um arquivo ao staging, em vez de todo o arquivo, para maior controle sobre o que será incluído no commit.

git add -p

.gitignore - Ignorando Arquivos

O arquivo .gitignore é usado para indicar ao Git quais arquivos ou pastas ele deve ignorar. Isso é útil para evitar adicionar arquivos de configuração local, logs, ou outros arquivos que não precisam ser versionados.
